# Авито - Кабинет продавца с AI-ассистентом

Веб-приложение - личный кабинет продавца с AI-ассистентом для улучшения объявлений


## Запуск

### Локально

```bash
# Сервер
cd server && npm install && npm start
# http://localhost:8080

# LLM (Ollama)
ollama pull llama3 && ollama serve

# Клиент
cd client && npm install && npm run dev
# http://localhost:5173
```

### Docker Compose

```bash
docker compose up --build -d
```

| Сервис | URL |
|--------|-----|
| Клиент | http://localhost:3000 |
| API | http://localhost:8080 |
| API через proxy | http://localhost:3000/api/* |

**Настройка LLM для Docker** — создайте `.env` в корне:

```env
VITE_LLM_BASE_URL=http://172.17.0.1:11434   # Linux
# VITE_LLM_BASE_URL=http://host.docker.internal:11434  # macOS/Windows
VITE_LLM_MODEL=llama3
```

## Команды

cd client

## Реализованные требования

Реализованы все обязательные и все дополнительные требования

### Дополнительные (со звёздочкой)

- **Чат с AI** - диалог с ассистентом в контексте конкретного объявления, история сообщений сохраняется в `localStorage`
- **Сравнение «Было - Стало»** - визуальный diff исходного и улучшённого описания с подсветкой изменений
- **Тёмная тема** - переключатель в шапке, выбор сохраняется в `localStorage`
- **Docker Compose** - одна команда для запуска клиента и сервера
- **Отмена запросов** - `AbortSignal` на всех API-запросах, запросы отменяются при переходе между страницами
- **Юнит-тесты** - 55 тестов покрывают компоненты, хуки, утилиты и API-слой

## Страницы

| Путь | Описание |
|------|----------|
| `/ads` | Список объявлений — поиск, фильтры, сортировка, пагинация |
| `/ads/:id` | Просмотр объявления |
| `/ads/:id/edit` | Редактирование с AI-ассистентом |

### AI-функции (страница редактирования)

- **Придумать / Улучшить описание** - генерирует текст, показывает сравнение «Было → Стало»
- **Узнать рыночную цену** - предлагает ценовой диапазон
- **Чат с AI** - контекст объявления передаётся автоматически

## Принятые решения

### 1. Поле `id` в ответе `GET /items`

В исходном API сервера отсутствовало поле `id` в ответе `GET /items`. Без него невозможно реализовать переход на страницу объявления. Поле добавлено в `server/data/items.json` и возвращается для каждого объявления

### 2. Сортировка по цене на клиенте

Сервер поддерживает сортировку только по `title` и `createdAt`. Сортировка по цене реализована на клиенте: при выборе «по цене» запрашиваются все подходящие объявления, сортируются в памяти и разбиваются на страницы. Для остальных колонок сортировка выполняется на сервере

### 3. Выбор Ollama для LLM

Для AI-функций используется локальная модель через Ollama. Модель `llama3` загружается один раз командой `ollama pull llama3`

### 4. Черновик формы в localStorage

При редактировании объявления изменения сохраняются в `localStorage` каждые 500 мс. При перезагрузке страницы черновик восстанавливается. После успешного сохранения - черновик очищается

### 5. Отмена запросов при переходе

Все API-запросы используют `AbortSignal` от TanStack Query. При переходе со страницы предыдущие запросы отменяются, чтобы избежать утечек памяти и гонок данных

### 6. UI-библиотека и структура

- Material UI выбран как наиболее зрелая и документированная библиотека
- Zustand для глобального состояния (тема) 
- TanStack Query + Axios для серверных запросов с кэшированием
- Vite как современная и быстрая система сборки

### 7. Тёмная тема

Реализована через кастомный стор на Zustand с сохранением выбора в `localStorage`. Переключатель находится в шапке приложения.
